# 🧠 DNS in Detail – Write-up

**Plataforma:** TryHackMe
**Trilha:** Intro to Offensive Security
**Dificuldade:** Iniciante
**Status:** ✅ Concluído

## 📌 Sobre o Lab

Este laboratório faz parte da trilha introdutória do TryHackMe e tem como objetivo ensinar, de forma prática, como o **DNS (Domain Name System)** funciona.

O foco não é exploração avançada, mas sim construir uma base sólida para reconhecimento em segurança ofensiva.

---

## 🎯 O que foi praticado

* Como funciona a resolução de nomes de domínio
* Diferença entre tipos de registros DNS (A, MX, CNAME, TXT)
* Uso de ferramentas básicas como `nslookup` e `dig`
* Interpretação de respostas DNS

---

## 🛠️ Ferramentas

* `nslookup`
* `dig`
* Terminal Linux

---

## 🚀 Abordagem

Realizei consultas DNS para entender como os domínios são traduzidos em endereços IP e como diferentes tipos de registros são retornados pelos servidores.

O lab guiado ajudou a reforçar conceitos essenciais que são utilizados na fase de **reconhecimento (recon)** em pentest.

---

## 🧩 Aprendizados

* DNS é uma das primeiras fontes de informação durante um reconhecimento
* Saber interpretar registros DNS é essencial antes de partir para exploração
* Mesmo labs básicos são importantes para construir fundamentos sólidos

---

## ⚠️ Observação

Este é um laboratório introdutório, então não envolve exploração real ou técnicas avançadas.
O foco aqui foi aprendizado de base — etapa essencial para evoluir em segurança ofensiva.
